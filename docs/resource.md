# 逆天改命算命软件资源要求文档

## 1. 文档说明

本文档用于梳理项目推进所需的资源要求，包括：

1. 开发机器资源
2. 测试与部署机器资源
3. 数据与存储资源
4. 账号与平台资源
5. 第三方服务资源
6. 权限与安全资源

本文档适用于项目从方案阶段进入实现、测试和上线准备阶段时的资源规划。

---

## 2. 资源规划目标

本项目不是单一 Web 页面，而是一个包含：

1. Web
2. Android
3. iOS
4. 后端 API
5. 异步任务
6. 图像分析
7. 数据存储
8. 通知与导出

的跨平台系统。

因此，资源规划必须覆盖：

1. 研发效率
2. 联调效率
3. 部署可行性
4. 上线可持续性

---

## 3. 研发机器资源

## 3.1 后端开发机要求

建议最低配置：

1. CPU：4 核
2. 内存：16 GB
3. 可用磁盘：100 GB SSD
4. 操作系统：macOS / Linux

建议配置：

1. CPU：8 核及以上
2. 内存：32 GB
3. 可用磁盘：200 GB SSD

原因：

1. 需要同时运行 FastAPI、PostgreSQL、Redis、Celery、MinIO
2. 本地可能需要处理图像分析依赖
3. 需要运行前端与接口联调环境

## 3.2 前端 Web 开发机要求

建议最低配置：

1. CPU：4 核
2. 内存：16 GB
3. 可用磁盘：50 GB SSD

建议配置：

1. CPU：8 核
2. 内存：16 GB 以上
3. 双显示器或高分辨率显示器

原因：

1. 需要同时运行 Next.js、设计稿、浏览器调试工具
2. UI 开发需要更好的显示面积

## 3.3 Android / iOS 开发机要求

### Android

建议最低配置：

1. CPU：8 核
2. 内存：16 GB
3. 可用磁盘：100 GB SSD

建议配置：

1. 内存：32 GB
2. 可用磁盘：200 GB SSD

原因：

1. Android 模拟器占用资源较高
2. React Native / Expo 构建和调试需要较稳定环境

### iOS

结论：

iOS 开发必须配备 `macOS` 机器。

建议配置：

1. 芯片：Apple Silicon
2. 内存：16 GB 起步，推荐 32 GB
3. 可用磁盘：150 GB SSD 以上

原因：

1. iOS 模拟器和 Xcode 依赖 macOS
2. 证书签名与真机调试都需要 Apple 开发环境

## 3.4 设计与产品机器要求

建议配置：

1. 内存：16 GB 以上
2. 可用磁盘：100 GB
3. 大屏显示设备

原因：

1. 需要查看原型、流程、文档和视觉设计
2. 多窗口协作效率更高

## 5. 测试与联调机器资源

## 5.1 测试环境服务器建议

建议最小配置：

1. CPU：4 核
2. 内存：8 GB
3. 磁盘：100 GB SSD

建议配置：

1. CPU：8 核
2. 内存：16 GB
3. 磁盘：200 GB SSD

用于运行：

1. Web
2. API
3. Redis
4. PostgreSQL
5. Celery Worker
6. MinIO

## 5.2 预发布环境服务器建议

建议配置：

1. CPU：8 核
2. 内存：16 GB 到 32 GB
3. 磁盘：200 GB SSD

原因：

1. 需要尽量接近生产环境
2. 需要做部署验证和性能压测

## 5.3 生产环境建议

早期生产环境建议至少拆成三类资源：

1. 应用节点
2. Worker 节点
3. 数据节点

### 应用节点建议

1. CPU：4 到 8 核
2. 内存：8 到 16 GB
3. 磁盘：100 GB

### Worker 节点建议

1. CPU：8 核
2. 内存：16 到 32 GB
3. 磁盘：100 GB

### 数据节点建议

1. CPU：8 核
2. 内存：16 到 32 GB
3. 磁盘：高性能 SSD，建议 200 GB 起

如果图像分析任务显著增多，应单独增加图像分析 Worker 节点。

## 6. 数据与存储资源

## 6.1 数据库资源

项目需要一个 `PostgreSQL` 实例。

职责：

1. 用户资料存储
2. 原始输入证据存储
3. 画像版本与变化记录
4. 历史人物结构化数据
5. 建议结果与反馈记录

### 存储建议

1. 开发环境：本地 PostgreSQL
2. 测试环境：独立测试实例
3. 生产环境：独立生产数据库

### 数据库运维要求

1. 自动备份
2. 迁移工具
3. 慢查询分析
4. 连接池配置

## 6.2 Redis 资源

项目需要一个 `Redis` 实例。

职责：

1. Celery Broker
2. 队列状态
3. 幂等控制
4. 缓存

### 最低要求

1. 测试环境可与应用同机
2. 生产环境建议独立实例或托管服务

## 6.3 对象存储资源

项目需要一个 `S3 兼容对象存储`。

职责：

1. 保存面部和手掌图片
2. 保存 PDF 报告
3. 保存分享海报
4. 保存中间导出文件

### 建议

1. 开发环境：MinIO
2. 生产环境：云对象存储

### 存储规划

建议至少拆分两个桶：

1. 原始上传桶
2. 导出与报告桶

## 16. 总结

本项目的资源要求重点不在单一服务器规模，而在于“跨平台 + 异步计算 + 图片处理 + 版本化数据 + 多账号体系”的组合复杂度。

如果只保留一句话：

要想顺利推进这个项目，至少要同时准备好：

1. 可支撑 Web + Android + iOS 的研发机器
2. PostgreSQL / Redis / 对象存储等基础设施
3. Apple / Google / Expo / 云平台 / AI 平台等账号资源
4. 基本的安全、备份、监控和发布能力
