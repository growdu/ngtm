# 逆天改命算命软件运维文档

## 1. 文档说明

本文档面向研发、运维和交付人员，说明系统部署、环境配置、日常运维、监控、备份、故障排查和版本发布要求。

当前文档基于现有产品与技术架构设计编写，适用于项目进入实现与部署阶段后的标准运维流程。

---

## 2. 系统概览

根据当前技术选型，系统主要包含以下组件：

1. Web 前端：`Next.js`
2. 移动端：`React Native + Expo`
3. 后端 API：`FastAPI`
4. 异步任务：`Celery Worker`
5. 定时任务：`Celery Beat`
6. 主数据库：`PostgreSQL`
7. 缓存与队列：`Redis`
8. 对象存储：`S3/MinIO`
9. 反向代理：`Nginx`

核心业务链路包括：

1. 用户建档
2. 八字分析
3. 图片上传与图像分析
4. 画像重算
5. 历史人物匹配
6. 改命建议生成
7. 通知和反馈回流

---

## 3. 环境规划

建议至少划分以下环境：

1. `dev`：开发环境
2. `test`：测试环境
3. `staging`：预发布环境
4. `prod`：生产环境

### 环境目标

#### dev

用于本地联调和功能开发。

#### test

用于接口测试、任务测试、图像分析流程测试。

#### staging

用于生产前验证，包括：

1. 配置验证
2. 数据迁移验证
3. 性能验证
4. 发布演练

#### prod

正式生产服务。

---

建议部署结构如下：


---

## 5. 服务器角色建议

在早期环境中，可使用简化部署：

1. 应用主机：Nginx + Next.js + FastAPI
2. Worker 主机：Celery Worker + Beat
3. 数据主机：PostgreSQL + Redis + MinIO

在生产环境中，建议至少将数据库与应用分离。

## 6. 配置管理

## 6.1 配置原则

1. 所有环境配置必须外部化。
2. 不允许把密钥写死在代码中。
3. 所有环境变量应区分开发、测试、预发布、生产。

## 6.2 关键配置项

建议至少管理以下配置：

1. `APP_ENV`
2. `APP_HOST`
3. `APP_PORT`
4. `DATABASE_URL`
5. `REDIS_URL`
6. `S3_ENDPOINT`
7. `S3_BUCKET`
8. `S3_ACCESS_KEY`
9. `S3_SECRET_KEY`
10. `JWT_SECRET`
11. `OPENAI_API_KEY` 或等价 AI 服务密钥
12. `CELERY_BROKER_URL`
13. `CELERY_RESULT_BACKEND`
14. `PUSH_NOTIFICATION_CONFIG`
15. `IMAGE_ANALYSIS_ENABLED`

## 6.3 配置存放建议

1. 本地环境：`.env.local`
2. 测试和预发布：CI/CD 密钥管理
3. 生产环境：密钥管理服务或安全配置中心

---

## 7. 数据库运维

## 7.1 数据库职责

PostgreSQL 存储：

1. 用户基础信息
2. 原始输入证据
3. 图片索引
4. 八字分析结果
5. 图像分析结果
6. 画像版本
7. 匹配结果
8. 建议计划与执行反馈

## 7.2 数据库迁移

建议使用：

1. `Alembic` 维护迁移脚本
2. 预发布环境先执行迁移演练
3. 生产迁移必须可回滚或有备份方案

## 7.3 数据库备份策略

生产环境建议：

1. 每日全量备份
2. 每小时增量或 WAL 归档
3. 保留最近 7 日每日备份
4. 保留最近 4 周每周备份

## 7.4 恢复演练

至少每月执行一次恢复演练，验证：

1. 备份文件可用
2. 恢复时间符合预期
3. 关键业务表可完整恢复

---

## 8. Redis 与任务队列运维

## 8.1 Redis 职责

Redis 用于：

1. Celery Broker
2. 任务状态缓存
3. 幂等控制
4. 热点查询缓存
5. 通知去重

## 8.2 关注指标

运维时重点关注：

1. 内存使用率
2. 键数量增长速度
3. 过期键清理情况
4. 队列积压长度
5. 响应延迟

## 8.3 队列策略

建议拆分至少三类队列：

1. `default`
2. `analysis`
3. `report_notification`

这样可以避免图像分析任务阻塞普通任务。

---

## 9. 对象存储运维

## 9.1 对象存储职责

对象存储用于保存：

1. 用户面部照片
2. 用户手掌照片
3. PDF 报告
4. 分享海报
5. 临时导出文件

## 9.2 运维要求

1. 启用私有桶或严格访问控制
2. 通过签名 URL 上传和下载
3. 设定临时文件清理策略
4. 为重要导出文件保留生命周期策略

## 9.3 安全要求

1. 禁止公开暴露原始敏感图片
2. 应区分原始图片桶与导出文件桶
3. 下载权限必须校验用户身份

## 11. 图像分析运维

## 11.1 图像分析职责

图像分析流程用于：

1. 面部关键点检测
2. 掌纹与手型检测
3. 图像质量评估

## 11.2 风险点

1. 图片过大导致处理超时
2. 图片质量差导致无有效特征
3. 图像分析任务堆积
4. 计算资源不足

## 11.3 运维建议

1. 上传前控制图片尺寸上限
2. 后端记录图像分析耗时
3. 将图像任务与普通任务分队列
4. 必要时独立扩容图像分析 Worker

## 12. 监控与告警

## 12.1 必须监控的系统指标

1. CPU
2. 内存
3. 磁盘
4. 网络流量

## 12.2 必须监控的应用指标

1. API 平均响应时间
2. API 错误率
3. Celery 队列长度
4. 画像重算成功率
5. 图片分析成功率
6. 匹配生成耗时
7. 建议生成耗时

## 12.3 必须监控的业务指标

1. 建档完成率
2. 问答继续率
3. 画像更新成功率
4. 人物匹配成功率
5. 建议反馈回流率

## 12.4 告警建议

建议在以下情况下触发告警：

1. API 5xx 持续升高
2. 数据库连接打满
3. Redis 不可用
4. Celery 队列积压超过阈值
5. 图片分析连续失败
6. 画像重算任务失败率过高

## 15. 常见故障与排查

## 15.1 用户上传图片失败

优先排查：

1. 对象存储配置
2. 签名 URL 是否有效
3. 上传文件大小限制
4. 前端跨域配置

## 15.2 八字分析没有生成

优先排查：

1. 基础信息是否完整
2. Celery 任务是否入队
3. Worker 是否在线
4. 规则库加载是否正常

## 15.3 画像没有更新

优先排查：

1. `recompute_profile` 任务是否触发
2. 上游分析结果是否成功写入
3. 数据库写入是否失败
4. 画像版本锁或幂等逻辑是否阻塞

## 15.4 历史人物匹配为空

优先排查：

1. 当前画像版本是否存在
2. 人物库数据是否导入
3. 匹配任务是否执行
4. 匹配阈值是否过高

## 15.5 建议没有生成

优先排查：

1. 是否已有有效画像版本
2. 匹配结果是否存在
3. `generate_advice` 任务是否成功
4. AI 文案服务或规则库是否异常

## 16. 数据安全要求

## 16.1 敏感数据范围

以下内容应视为敏感数据：

1. 出生信息
2. 面部与手掌照片
3. 行为问答
4. 人生事件
5. 画像结果

## 16.2 安全要求

1. 全链路使用 HTTPS
2. 数据库存储应限制访问来源
3. 敏感对象存储禁止公开访问
4. 密钥应定期轮换
5. 生产环境操作必须有审计记录
