# 逆天改命算命软件实施路线文档

## 1. 文档说明

本文档用于定义项目从方案到 MVP、再到多端增强和生产化的实施路线。

目标是回答三个问题：

1. 先做什么
2. 后做什么
3. 每个阶段的完成标准是什么

---

## 2. 路线设计原则

实施顺序遵循以下原则：

1. 先跑通核心业务闭环，再扩展功能面
2. 先保证可解释和版本化，再追求复杂算法
3. 先完成 Web + 后端主链路，再增强 Android / iOS / Desktop
4. 先做规则驱动 MVP，再评估更复杂模型

---

## 3. 总体阶段划分

建议分为六个阶段：

1. 阶段 0：工程准备
2. 阶段 1：MVP 主闭环
3. 阶段 2：持续画像增强
4. 阶段 3：移动端与通知能力
5. 阶段 4：桌面端与报告能力
6. 阶段 5：生产化与扩展能力

---

## 4. 阶段 0：工程准备

### 目标

把项目从文档阶段推进到可开发状态。

### 主要工作

1. 确定最终技术方案
2. 搭建 monorepo
3. 初始化 Next.js / FastAPI / React Native 工程骨架
4. 初始化 PostgreSQL / Redis / MinIO 本地环境
5. 建立 CI 基础流程
6. 建立代码规范、提交规范和环境变量模板

### 交付物

1. 项目目录结构
2. 基础脚手架
3. 本地启动说明
4. 环境变量模板

### 完成标准

1. 仓库可安装依赖
2. Web、API、Worker、数据库可本地启动
3. 移动端骨架可运行

---

## 5. 阶段 1：MVP 主闭环

### 目标

完成“证据 -> 初始画像 -> 持续问答 -> 新画像 -> 历史人物匹配 -> 建议输出”的最小闭环。

### 范围

1. 用户建档
2. 八字分析
3. 持续问答
4. 画像版本化
5. 历史人物匹配
6. 建议生成
7. 基础成长档案

### 主要工作

1. 建立数据库表结构
2. 实现基础建档接口
3. 实现八字分析模块
4. 实现问答模块
5. 实现画像引擎 V1
6. 实现人物匹配 V1
7. 实现建议模块 V1
8. 实现 Web 端主流程页面

### 交付物

1. MVP API
2. Web MVP 页面
3. 画像版本生成链路
4. Top 3 历史人物匹配结果
5. 基础建议页

### 完成标准

1. 用户能完成首次建档
2. 用户能获得 V1 画像
3. 用户回答问题后能触发 V2 画像
4. 系统能给出匹配人物和建议

---

## 6. 阶段 2：持续画像增强

### 目标

让系统真正具备“持续演进”的特性。

### 范围

1. 人生事件录入
2. 画像变化说明
3. 画像版本对比
4. 更精细的建议回流
5. 更丰富的人物画像库

### 主要工作

1. 实现 life event 输入链路
2. 完善 profile change log
3. 增强匹配解释
4. 增强建议反馈机制
5. 优化问答策略

### 完成标准

1. 用户新增事件后画像可变化
2. 用户能查看画像版本差异
3. 匹配结果能解释相似点和差异点

---

## 7. 阶段 3：移动端与通知能力

### 目标

让 Android / iOS 具备核心使用能力。

### 范围

1. React Native 页面
2. 登录和建档
3. 持续问答
4. 事件记录
5. 建议提醒
6. 图片拍照与上传

### 主要工作

1. 实现 Expo 工程
2. 实现移动端建档与问答页面
3. 接入拍照和相册上传
4. 接入推送与提醒
5. 接入移动端 API SDK

### 完成标准

1. Android / iOS 可完成建档
2. Android / iOS 可参与持续问答
3. Android / iOS 可接收提醒

---

## 8. 阶段 4：桌面端与报告能力

### 目标

补足桌面端与导出能力。

### 范围

1. Tauri 桌面端
2. PDF 导出
3. 分享海报
4. 长文本事件录入

### 主要工作

1. 实现 Tauri 壳层
2. 实现导出接口与前端页面
3. 优化成长档案展示

### 完成标准

1. 桌面端可运行核心流程
2. 用户可导出 PDF 报告
3. 分享文件生成可用

---

## 9. 阶段 5：生产化与扩展能力

### 目标

为正式上线和中期增长做准备。

### 范围

1. 图像分析增强
2. 监控与告警
3. 容量规划
4. Worker 扩容
5. 数据备份与恢复
6. 安全治理

### 主要工作

1. 图像分析服务化评估
2. 推进完整 CI/CD
3. 完善部署与运维工具
4. 补充压测和故障演练

### 完成标准

1. 系统具备上线条件
2. 关键组件具备监控与告警
3. 可完成回滚和恢复演练

---

## 10. 阶段优先级总结

最重要的阶段顺序是：

1. 阶段 0：工程准备
2. 阶段 1：MVP 主闭环
3. 阶段 2：持续画像增强
4. 阶段 3：移动端与通知能力

原因：

1. 产品核心价值在持续画像，不在图像炫技
2. 必须先验证画像演进与人物匹配是否成立
3. 移动端应该在主闭环稳定后快速接入

---

## 11. 里程碑建议

建议设定以下里程碑：

### M1：工程可开发

完成：

1. Monorepo 初始化
2. 基础服务启动
3. 开发环境就绪

### M2：Web MVP 可用

完成：

1. 建档
2. 初始画像
3. 问答校准
4. 匹配
5. 建议

### M3：持续画像成立

完成：

1. 事件回流
2. 版本变化
3. 成长档案

### M4：移动端闭环可用

完成：

1. Android / iOS 建档
2. Android / iOS 问答
3. Android / iOS 提醒

### M5：生产化准备完成

完成：

1. 部署
2. 监控
3. 备份
4. 发布流程

---
