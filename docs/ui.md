# 逆天改命算命软件 UI 原型设计与交互流程文档

## 1. 文档目标

本文档定义产品的 UI 界面原型、页面结构、核心组件和主要交互流程，作为后续视觉设计、前端开发和产品评审的统一依据。

目标不是做一个“静态算命报告页”，而是设计一个会持续追问、持续修正、持续成长的命理画像产品界面。

---

## 2. UI 设计目标

### 2.1 产品界面定位

界面气质应当接近：

1. 神秘但不廉价
2. 东方命理感，但不做传统黄历站风格
3. 数据化、可信、可追踪
4. 强调“画像演进”和“人物映射”，而不是一次性判词

### 2.2 设计关键词

1. 命盘感
2. 档案感
3. 仪式感
4. 演进感
5. 可解释性

### 2.3 视觉方向

建议采用“深墨色底 + 暖金/朱砂点缀 + 骨白文字卡面”的视觉体系。

建议基调：

1. 背景：深青黑、墨蓝黑、局部星图纹理
2. 主强调色：古金、铜金
3. 次强调色：朱砂红、青玉绿
4. 内容面板：半透明深色层或温润浅卡面
5. 图形语言：圆盘、轨迹、轴线、卷轴、印章、命格雷达

避免：

1. 传统玄学站的廉价金龙、火焰特效
2. 大量发光边框
3. 紫色科技风模板
4. 通用 SaaS 卡片堆砌

## 4. 全局布局原型

### 4.1 桌面端布局

建议使用三段式布局：

1. 顶部全局导航栏
2. 左侧章节导航/进度轴
3. 中央主内容区
4. 右侧辅助栏

右侧辅助栏用于展示：

1. 当前画像版本号
2. 结论置信度
3. 本轮补充后会影响的维度
4. 今日提醒

桌面端结构：

```text
+----------------------------------------------------------------------------------+
| Logo | 首页 | 初始分析 | 画像演进 | 历史人物 | 改命建议 | 成长档案 | 用户头像 |

| 左侧导航/进度轴 |                 主内容区                         | 右侧信息栏 |
|                |                                                   |           |
| 1. 建档        | 页面标题                                           | 当前版本V4 |
| 2. 上传        | 页面说明                                           | 置信度78%  |

| 4. 追问        |                                                   | 今日提醒   |

```

3. 纵向内容流

底部导航建议保留：

1. 首页
2. 画像
3. 历史人物

## 5. 核心页面原型

## 5.1 首页原型

### 页面目标

让用户理解这不是一次性算命，而是“持续理解你、逐步接近真相”的系统。

### 页面模块

1. 顶部主标语
2. 开始建档按钮
3. 当前画像摘要卡
4. 最新人物匹配卡
5. 今日建议卡
6. 成长进度时间线入口

### 原型草图

```text
+--------------------------------------------------------------------------------+
| 你的命，不止能算，还能被持续校正                                               |
| 从出生信息、照片、经历到性格模式，逐步形成你的命运画像                         |
| [开始建档]   [查看当前画像]                                                    |
+--------------------------------------------------------------------------------+
| 当前画像摘要         | 最像的历史人物       | 今日建议                         |
| V4 画像              | 曹操 82%             | 今日宜：整理卧室东南角           |
| 风险偏好高           | 相似点：权力驱动强   | 忌：冲动投资                     |
| 情绪稳定性中         | 差异：情感克制更强   | [去执行]                         |
+--------------------------------------------------------------------------------+
| 最近演进                                                                   >   |
| V3 -> V4：系统认为你比之前更偏向长期主义                                       |
+--------------------------------------------------------------------------------+
```

### 关键交互

1. 点击“开始建档”进入建档流程。
2. 点击摘要卡进入对应详情页。
3. 首页应始终展示“上次更新后发生了什么变化”。

## 5.3 照片上传页原型

### 页面目标

引导用户提供高质量面相和手相图片，同时降低上传焦虑。

### 页面模块

1. 面部照片上传卡
2. 手掌照片上传卡
3. 拍摄规范示意
4. 图像质量状态

### 原型草图

```text
+--------------------------------------------------------------------------------+
| 第二步：补充外在特征                                                           |
| 你的照片不会单独决定结论，它们只会作为画像的辅助证据                           |
+--------------------------------------------------------------------------------+
| 面部照片                               | 手掌照片                               |
| [上传正脸照]                           | [上传左手掌]                           |
| 光线均匀、无遮挡                       | 掌心朝上、手指自然张开                 |
| 质量：待上传                           | 质量：待上传                           |
+--------------------------------------------------------------------------------+
| 拍摄提示                                                                      |
| 1. 正脸直视镜头                                                                |
| 2. 不使用夸张滤镜                                                              |
| 3. 手掌线条需清晰                                                              |
+--------------------------------------------------------------------------------+
| [稍后上传]                                  [继续]                             |
+--------------------------------------------------------------------------------+
```

### 关键交互

1. 上传后即时显示质量评分。
2. 质量不足时只显示“建议重拍”，不直接给低质结论。
3. 支持先跳过，后续在“我的资料”补传。

### 页面模块

1. 初始命盘摘要
2. 第一版性格推断
3. 当前置信度
4. 不确定项
5. 下一轮问题入口

### 原型草图

```text
+--------------------------------------------------------------------------------+
| 你的第一版命理画像 V1                                                          |
| 当前结论置信度：61%                                                            |
+--------------------------------------------------------------------------------+
| 命盘摘要                                                                       |
| 年柱：甲子  月柱：戊辰  日柱：辛未  时柱：待校准                               |
| 五行：金偏强，木偏弱                                                           |
+--------------------------------------------------------------------------------+
| 初步推断                                                                       |
| - 行动力强，但做重大决定时偏保守                                               |

| - 事业驱动高于情感驱动                                                         |
```

### 页面目标

把问答变成“画像校准过程”，而不是冷表单。

### 页面模块

1. 当前问题卡
2. 问题影响说明
3. 回答方式切换
4. 当前进度
5. 即时变化提示

### 原型草图

```text
+--------------------------------------------------------------------------------+
| 校准你的画像：第 3 / 8 题                                                      |
| 这道题将帮助系统判断：风险偏好、长期主义、事业驱动力                           |
+--------------------------------------------------------------------------------+
| 当你面对一个高收益但高不确定性的机会时，你通常会？                             |
|                                                                                |
| ( ) 快速抓住                                                                   |
| ( ) 观察一段时间后再决定                                                       |
| ( ) 只有非常确定才行动                                                         |
| ( ) 视当时状态而定                                                             |
|                                                                                |

+--------------------------------------------------------------------------------+
| 当前回答后，系统可能会：                                                       |

| - 调整“人物匹配”候选                                                           |
```

2. 回答后可弹出微反馈：“你比上一版更偏向长期主义”。
3. 支持文字补充，以便后续 LLM 提取更细证据。

## 7. 关键状态设计

### 7.1 页面状态

每个核心页面都要覆盖：

1. 初始空状态
2. 加载中状态
3. 成功状态
4. 部分数据缺失状态
5. 错误状态

### 7.2 特殊状态

#### 出生时辰不确定

展示方式：

1. 横幅提示“当前为多候选命盘推断”
2. 提供“继续校准”按钮

#### 图片质量不足

展示方式：

1. 说明问题原因
2. 给出重拍示意
3. 不直接输出强结论

#### 画像冲突较大

展示方式：

1. 用“低置信度”标识
2. 给出“为什么系统还不确定”

## 8. 主交互流程

## 8.1 首次使用主流程

```text
进入首页
-> 点击开始建档
-> 填写基础信息
-> 上传照片或跳过
-> 系统生成 V1 初始画像
-> 查看初始分析
-> 进入持续问答
-> 生成 V2 画像
-> 进入历史人物匹配页
-> 查看第一版建议
```

### 交互目标

1. 让用户尽快拿到第一版结果
2. 在拿到结果后继续诱导补充信息
3. 尽快让用户看到“画像变化”和“人物匹配”

## 8.2 持续校准流程

```text
用户进入画像页
-> 看到“还有 3 项关键不确定信息”
-> 进入问答页补充
-> 系统重算画像
-> 弹出“你的长期主义判断上升”
-> 人物匹配从王安石切换为曹操
-> 用户继续查看原因
```

### 交互目标

1. 用户感知到“我输入的信息真的改变了结果”
2. 每次变化都有解释

## 10. 交互细节规范

### 10.1 动态反馈原则

系统在以下操作后必须给即时反馈：

1. 提交问题答案后
2. 上传图片后
3. 记录重大事件后
4. 执行建议反馈后

反馈形式：

1. 小浮层
2. 页面顶部更新条
3. 版本变化卡

示例文案：

1. “系统提高了你对长期关系稳定性的判断”
2. “最新经历让你更接近王安石型人格，而非韩非型”

### 10.2 不确定性表达规范

禁止：

1. 绝对命令式文案
2. 100% 确定式判断

建议表达：

1. 当前推断
2. 倾向于
3. 证据较强
4. 仍需校准
5. 当前置信度

### 10.3 动画建议

建议使用少量但明确的动效：

1. 画像重算时使用轨迹汇聚动画
2. 人物匹配切换时使用卷轴展开或轮盘切换
3. 时间线版本切换时使用平滑过渡

避免：

1. 花哨粒子特效
2. 长时间等待动画
