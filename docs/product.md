

# 《逆天改命算命软件》产品需求文档（PRD）

## 1. 产品概述

* **产品名称**：逆天改命算命软件
* **产品类型**：持续交互式命理分析 + 运势优化 + 人物画像工具
* **目标用户**：

  * 对周易、八字、五行、风水、面相、手相、人格画像感兴趣的人群
  * 希望在长期互动中逐步获得更准确结论、决策参考和改善建议的用户
  * 希望理解“自己像谁”“自己会如何演化”的用户
* **核心价值**：

  1. 不是一次性出结果，而是随着信息增多持续修正结论
  2. 将命理、外貌、行为、经历整合成动态用户画像
  3. 基于画像匹配历史上最相似的人物，增强理解与传播性
  4. 提供可执行的逆天改命建议，并追踪改善效果

---

## 2. 产品定位升级

本产品不应设计成传统“填一次信息，生成一次报告”的工具，而应设计成持续交互演进系统。

系统核心逻辑：

1. 用户先提供少量基础信息，获得初始推断
2. 系统根据当前不确定项继续追问
3. 用户补充新的资料、经历、偏好、照片、事件反馈
4. 系统持续调整对性格、能力、运势、风险、优势的判断
5. 最终沉淀为长期稳定的用户画像
6. 在画像成熟后，匹配历史人物原型，并解释相似点与差异点

这意味着产品输出不再是静态报告，而是一个会成长、会修正、会解释不确定性的“命理人格档案”。

## 3. 产品功能

### 3.1 分阶段信息采集

| 阶段 | 信息类型 | 示例 | 输入方式 |
| ---- | ---- | ---- | ---- |
| 初始阶段 | 基础信息 | 出生日期、出生时辰、性别、出生地 | 日期选择器 + 下拉菜单 |
| 增强阶段 | 图像信息 | 面部照片、手掌照片 | 文件上传 / 手机拍照 |
| 校准阶段 | 行为与经历 | 职业、感情状态、重大转折、家庭结构、健康习惯 | 问答表单 / 对话输入 |
| 深化阶段 | 心理与偏好 | 决策方式、价值观、社交偏好、金钱观、风险偏好 | 多轮问答 |
| 演进阶段 | 持续反馈 | 实际发生事件、每月状态、自评变化、建议执行结果 | 时间线记录 / 打卡 / 对话更新 |

要求：

* 每次采集都要明确“本轮新增信息将修正哪些判断”
* 允许用户跳过问题，但系统应记录“不确定性”
* 系统应优先追问对结论影响最大的缺失信息

---

### 3.2 动态八字分析

* **功能**：

  * 四柱计算（年柱、月柱、日柱、时柱）
  * 五行强弱统计
  * 十神关系分析（比肩、劫财、食神、伤官、财星等）
  * 大运流年预测
  * 出生时辰不确定时的多候选推断
  * 结合用户反馈对命格解释做置信度调整
* **输出**：

  * 命格类型与性格分析
  * 财运、事业、婚姻、健康运势预测
  * 结论置信度
  * 与后续反馈冲突的假设项

说明：

* 八字结论不是固定文案，应允许因用户后续经历反馈而修正解释层。
* 原始命盘不变，但“命盘解释优先级”和“现实映射结论”可以演进。

---

### 3.3 面相分析

* **功能**：

  * 面部关键点检测（眉、眼、鼻、口、下巴）
  * 五官结构特征量化
  * 对应运势与性格解释
  * 多张照片交叉校验，降低单次拍摄误差
* **输出**：

  * 五官吉凶解读
  * 性格与能力倾向
  * 运势提示（事业、婚姻、健康）
  * 图像质量评分与识别可信度

---

### 3.4 手相分析

* **功能**：

  * 掌纹检测（生命线、智慧线、感情线、事业线）
  * 手型判断（方手、尖手、混合手）
  * 手指长度、指节分析
  * 左右手差异分析
* **输出**：

  * 健康、寿命倾向判断
  * 性格与能力倾向
  * 事业与婚姻运势
  * 当前阶段判断与历史采样差异

---

### 3.5 性格与人生轨迹校准模块

这是新增核心模块，用于把命理推断从“玄学解释”推进到“可持续修正的人格画像”。

* **功能**：

  * 基于多轮问答生成人格维度
  * 收集人生关键事件并识别模式
  * 分析冲突处理方式、情感依恋模式、事业驱动力、风险偏好、权力欲、创造欲、秩序感
  * 根据新证据持续修正对性格和人生走向的判断
* **输出**：

  * 性格画像
  * 能力结构画像
  * 关系模式画像
  * 财富与事业驱动画像
  * 人生阶段演进描述

建议画像维度：

* 内向/外向
* 感性/理性
* 冲动/克制
* 稳定/冒险
* 依附/独立
* 权威导向/自由导向
* 秩序型/创造型
* 长线主义/即时满足

---

### 3.6 综合评分与动态画像

旧设计中的一次性综合评分需要升级为“动态画像引擎”。

建议评分结构：

* 八字：40%
* 面相：15%
* 手相：15%
* 行为经历校准：20%
* 多轮人格问答：10%

输出内容：

* 综合运势评分（0-100）
* 性格画像评分矩阵
* 能力雷达图
* 当前阶段关键词
* 主要优劣势提示
* 未来关键年份提醒
* 当前版本画像编号
* 与上一版画像的变化说明

系统必须支持：

* 每次新增信息后重新计算画像
* 标注哪些结论被增强、削弱或推翻
* 保留版本历史，便于用户查看“我被系统如何逐步理解”

---

### 3.7 历史人物匹配模块

这是新增差异化功能，用于将用户画像映射到历史人物原型。

* **功能**：

  * 建立历史人物画像库
  * 对历史人物进行结构化标签整理
  * 将用户画像与人物画像进行多维匹配
  * 给出“最匹配人物”及 Top N 候选
* **人物画像维度示例**：

  * 性格结构
  * 权力倾向
  * 财富观
  * 情感模式
  * 行动力
  * 风险偏好
  * 命运起伏节奏
  * 成功方式与失败模式
* **输出**：

  * 最匹配历史人物
  * 相似度分数
  * 相似点解释
  * 差异点解释
  * 为什么不是其他候选人
  * “你更像某人物的哪个人生阶段”

设计要求：

* 历史人物匹配不能只靠一句文案，要显示可解释的维度匹配结果
* 人物库需支持帝王、谋士、将领、商人、文人、改革者、枭雄、隐士等类别
* 匹配结果应随画像演进而变化

---

### 3.8 逆天改命模块

* **功能**：

  1. 风水布局建议（家居、办公室、房间色彩、五行摆件）
  2. 行为/习惯改善方案（投资、健康、心态、技能提升）
  3. 吉日吉时提醒（签约、出行、结婚等重要事务）
  4. 画像弱项修正建议（例如优柔寡断、情绪化、关系依赖、执行力不足）
  5. 基于历史人物原型给出“进化方向建议”
* **输出**：

  * 文本 + 图示 + 可保存 PDF
  * 日历提醒（手机通知 / 桌面通知 / 浏览器通知）
  * 执行建议后的追踪任务

要求：

* 建议必须与画像直接关联，而不是通用模板
* 后续应追踪建议执行后是否带来结果改善

---

### 3.9 历史记录、版本对比与成长档案

* 保存用户每一轮分析数据
* 对比不同时间的运势评分变化
* 对比不同版本的人格画像变化
* 记录用户提供的新证据如何影响判断
* 收藏和分享功能
* 生成“个人命运演进档案”

## 4. 多端支持

| 平台 | 功能实现 |
| ---- | ---- |
| 手机端 | Flutter/React Native；摄像头拍照；本地通知；持续问答；事件记录；本地保存分析结果 |
| 桌面端 | Electron/Tauri；外接摄像头支持；桌面通知；本地文件保存；长文本录入；报告导出 |
| Web端 | React/Next.js；MediaDevices API 调用摄像头；浏览器通知；PDF 导出；时间线与画像可视化 |

**统一策略**：

* 前端共享 UI 组件和数据渲染逻辑
* 后端统一提供 API（REST/GraphQL）
* 数据存储统一 PostgreSQL + JSONB
* 画像引擎与人物匹配引擎由统一服务提供

## 6. 技术实现

* **后端**：Python（FastAPI 优先），提供统一 API
* **前端**：

  * Web：React + ECharts
  * 手机端：Flutter / React Native
  * 桌面端：Electron / Tauri
* **图像处理**：OpenCV + DNN 或 MediaPipe
* **可视化**：雷达图、柱状图、画像漂移图、时间线、面相手相关键点标注
* **算法分层**：

  * 八字运算层：天干地支 + 五行 + 十神规则库
  * 图像分析层：关键点检测 + 规则映射
  * 画像引擎层：多源证据融合 + 置信度更新 + 版本化
  * 人物匹配层：历史人物结构化标签 + 向量化匹配/规则匹配
  * 建议生成层：规则库 + AI 生成文本

建议实现原则：

* 规则计算与 AI 文案分离
* 所有关键结论都要有来源和置信度
* 用户画像要可回溯、可版本化、可解释

---

## 7. 用户界面概念

1. **首页**：基础信息输入 + 上传照片 + 分析深度选择
2. **初始结论页**：先输出第一版结论，并明确哪些地方还不确定
3. **持续问答页**：系统追问关键问题，用户逐步补充信息
4. **动态画像页**：展示性格、能力、关系、事业、运势的当前画像
5. **历史人物匹配页**：展示最匹配人物、候选列表、相似与差异解释
6. **改命建议页**：风水布局图 + 行为改善列表 + 吉日日历 + 执行追踪
7. **成长档案页**：画像版本演进、关键事件时间线、历史报告对比

---

## 8. 核心功能优先级

1. 八字分析 + 初始结论生成
2. 持续问答机制 + 动态画像版本化
3. 面相手相分析接入画像引擎
4. 历史人物画像库 + 人物匹配模块
5. 逆天改命建议与执行追踪
6. 多端完整支持 + 通知 + PDF/分享

---

## 9. MVP 建议

第一阶段不建议一开始就把所有玄学模块做满，优先做出“会成长的画像系统”。

MVP 范围：

* 基础信息录入
* 八字分析
* 多轮问答
* 用户画像版本化
* 简版历史人物匹配
* 基础建议输出

这样可以先验证两个核心价值：

1. 用户是否愿意持续补充信息
2. “你最像哪位历史人物”是否形成强传播与留存

---

## 10. 产品成功标准

* 用户完成初始分析后的继续补充率
* 用户平均完成的画像版本数
* 历史人物匹配页的分享率
* 用户对“画像是否像我”的主观认可度
* 建议执行后的回访率
* 7 日与 30 日留存
